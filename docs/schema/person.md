# Person Schema

## Purpose
The person schema is the canonical data contract for all genealogy records. Every person is stored as a Hugo page bundle at:
- `src/content/family/<person_id>/index.md`

The front matter must follow the schema exactly. The admin UI and API validate all writes against this contract.

## Field Overview

### Identity
- `title`: Display name (mirrors `names.full`).
- `date`: Metadata timestamp (ISO string, full date/time).
- `draft`: Hugo draft flag.
- `person_id`: Stable primary key (UUID4, immutable once created). This is not exposed in the UI.
- `slug`: Optional canonical slug hint.
- `aliases`: Legacy slugs for redirects.
- `names`: Structured name data.
  - `full`, `given`, `middle`, `surname`, `suffix`, `maiden`, `also_known_as`.
- `sex`: `M`, `F`, or `U`.

### Vitals
- `vitals.born` and `vitals.died` accept partial ISO dates: `YYYY`, `YYYY-MM`, or `YYYY-MM-DD`.
- `birth_place`, `death_place`, `burial_place`, `cause_of_death` are free text.

### Story
- `story_md` stores the long-form person narrative (Markdown).
- Authored in admin WYSIWYG and rendered as markdown on the website.

### Relationships
- `relations.parents.father` / `relations.parents.mother`: `person_id` references.
- `relations.spouses`: list of spouse entries with dates/places.
- `relations.children`: list of `person_id` values.
- `relations.siblings`: optional manual overrides.

### Media
All media files must be bundle-relative paths and must live inside the bundle, typically under `gallery/`.
- `media.featured`: Primary portrait / featured image path (used for tree cards and person headers).
- `media.gallery`: list of media items with metadata and `source_key` references.
- `timeline[*].media`: per-event image/document entries for timeline rendering.

### External IDs
`ids` contains optional references (Find a Grave, FamilySearch, WikiTree, etc.).

### Evidence / Sources
`sources` is a list of reusable evidence records. `media.gallery[*].source_key` references `sources[].key`. Keys must be unique within a record.

### Confidence
A structured way to record certainty: `identity`, `vitals`, `parents`, plus free-form `notes`.

### Provenance
Tracks the origin of the record, e.g. WordPress import.

### Timeline
- `timeline[]` stores person-specific timeline entries.
- Each event supports: `start_date`, `end_date`, `title`, `event_type`, `location`, `story_md`, `media[]`, `source_refs[]`, `related_people[]`, `sort_weight`.
- Event dates accept partial ISO format.
- If both event dates exist, `end_date` must not be earlier than `start_date`.

## Relationship Modeling
- All relationship values are stored as `person_id` references.
- Parent-child relationships must not be cyclical.
- Self-parenting is invalid.
- Spouse entries must not be duplicated.
- Sibling overrides are allowed but should be used sparingly.

## Hidden ID Strategy
- `person_id` is generated by the system using UUID4.
- Users never create, edit, or view raw IDs in the UI.
- Relationship fields present human-friendly display names and store UUIDs internally.

## Media Bundling Rules
- Files must be stored inside the person bundle.
- Paths must be bundle-relative.
- The admin tool prevents filename collisions by suffixing uploads.

## Folder Naming
- Person bundles are stored under `src/content/family/<uuid>[-slug]/index.md`.
- The folder name must start with the UUID, with an optional slug suffix.

## Source Linking Philosophy
Sources are structured once and referenced by key, so media items can share provenance.

## Genealogy Uncertainty
Confidence fields provide a structured way to mark uncertain identity, vitals, or parents without losing context.


## Admin Editing Layout
The admin editor is organized to match the schema workflow:
- **Story**: `story_md` for the person-level narrative.
- **Events**: `timeline[]` entries with date range, title/type/location, event story, optional event media path/caption, source refs, and related people.
- **Gallery**: `media.featured` and `media.gallery[]` for bundle media assets.

When saving a person, timeline events are sorted by date and shown in a date-ordered list in the editor.
