# Project Philosophy

## File-First Architecture
All data is stored as plain files in `src/`. This makes the system transparent, inspectable, and easy to back up.

## No Database Requirement
The stack runs without a database. This reduces operational complexity and keeps data durable across decades.

## Long-Term Data Durability
Genealogy data should outlive specific tools. File-backed storage ensures portability.

## Transparency of Genealogy Data
Edits and relationships are stored in human-readable formats to support auditing and long-term stewardship.
